# 黑鸦标题风格

本技能用于把主题、事件、产品发布、争议、比赛结果或新闻素材改写成黑鸦式中文视频标题：信息密度高、情绪强、实体明确、多个事件压缩在同一个标题里。

## 引用资料

使用本技能前，优先查看并吸收以下引用资料，再开始改写标题：
1. `references/02-style-analysis.md`：人类可读的风格分析报告，先用它理解整体风格、结构比例、情绪强度和代表标题。

2. `references/02-style-analysis.yaml`：结构化分析数据（YAML），用于精确读取长度分布、词频、实体、短语、情绪强度、开头/结尾钩子和时间趋势。

3. `references/01-titles.csv`：原始标题样本，CSV 格式（aid,bvid,title,created,createdDate），用于在需要时查找真实标题语感、标点节奏和近期用词。

如果引用资料和下方摘要有差异，以 `references/` 中的最新资料为准。不要只凭本文件里的摘要生成标题；摘要负责快速定位，引用资料负责校准细节。

## 核心特征

- 标题长度：平均 53.9 字，P75 64 字，P90 72 字；82.9% 超过 40 字。
- 信息密度：平均 4.4 个分句，平均 4.16 个分隔符。
- 情绪强度：平均情绪分 2.74；18.6% 以感叹号收尾，14% 使用疑问钩子。
- 事实钩子：81.4% 带数字，83.7% 带品牌、产品或人物名。
- 叙事倾向：冲突/对比 3.9%，合集/日报 65.9%。
- 主导结构：「情绪爆点」占 89.9%，常与情绪词、数字、实体名组合。

写作时把标题当成压缩新闻戏剧：实体名 + 突发动作 + 行业后果 + 情绪标点。

## 标题公式

优先使用以下公式：

1. `{实体/公司} + 突然/重磅/炸裂 + {动作}，{对手/行业} + {反应/后果}！`
2. `{事件A}；{事件B}；{事件C} | AI日报MMDD`
3. `{数字/首次/万亿} + {事实突破}，{旧格局} 要被改写？`
4. `{人物/公司} 硬刚 {对手}，{结果} 太离谱了！`
5. `{产品/模型} 发布后，{行业/用户/资本市场} 全变了`

样本结构占比：
- 情绪爆点: 89.9%
- 数字悬念: 81.4%
- 日报式: 65.9%
- 多事件合并: 31.8%
- 对比冲突: 3.9%
- 其他: 0.8%

常见开头钩子：重磅炸弹、见证历史、头皮发麻、AI圈大地震、后背发凉、突发、卧槽、AI奇点已至、AI圈已沸腾、【黑鸦Heya】一个视频带你看完Go、180天断供倒计时、阿里千问正式发布Qwen3.6-Pl
常见结尾钩子：AI日报、战啊……、AI日报3月5日、【黑鸦Heya】一个视频带你看完Go、奥特曼必须立刻移交军事法庭……、百万程序员的斩杀线来了……、达里奥吓到冷汗直流、发起挽救人类倡议、高价老用户的斩杀线来了、更加自然流畅、谷歌开源模型 Gemma 4 即将发、或将发布
常见分隔符：！、，、|、？、；、：、、、｜

## 写作规则

1. 默认写中文，除非用户明确要求其他语言。
2. 优先输出一个长标题；用户要多个时，再给 3-5 个强度不同的版本。
3. 标题必须保留用户给出的事实，不编造新公司、新数据、新结论。
4. 先放最强冲突或最大结果，再用逗号、分号、竖线串联第二第三事件。
5. 情绪词服务于事实：可以更戏剧化，但不能把不确定信息写成确定事实。
6. 如果输入是日报、周报、合集，保留 `| AI日报MMDD` 或同类收束标签。
7. 如果输入偏产品发布，突出“发布/上线/开源/升级 + 行业反应”。
8. 如果输入偏争议事件，突出“谁对谁 + 冲突动作 + 代价/后果”。
9. 避免短平标题；少用泛泛的“来了”“看看”，多用具体实体、数字和后果。
